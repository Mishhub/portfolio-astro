## Context

A confidential Web3 ecosystem spanned several products that each needed the same
things: one identity a user signs in with everywhere, KYC-gated access, a wallet
that's pleasant to use on-chain, and a referral/reward program that pays out
fairly. The work was to build that shared backbone — identity, wallets and rewards —
as services the products could all build on.

## Architecture

A centralized **OAuth2 / OpenID Connect** service is the identity backbone; a
**NestJS** wallet gateway abstracts chain interactions; and services talk to each
other over **gRPC** for real-time sync. JWTs and role-based access control carry a
scoped identity across every product.

## Centralized identity

Single sign-on is built on **OAuth2 + OpenID Connect**, so a user authenticates once
and every product trusts the same identity. **KYC (via SumSub)** feeds **dynamic
access levels** — what a verified user can do differs from an unverified one — and
**JWT** sessions plus **RBAC** enforce that consistently across services.

## The wallet gateway

A dedicated **NestJS + Viem** microservice fronts all blockchain interactions, so
products never touch RPC details directly. **Account Abstraction** via **Biconomy**
and **Alchemy** makes the wallet experience smoother — smart-account flows instead of
raw externally-owned-account friction.

<Callout type="insight" title="A wallet gateway is an anti-corruption layer">
  Putting every chain interaction behind one NestJS service means RPC providers,
  gas strategy and account-abstraction details can change without rippling through
  the products that depend on it.
</Callout>

## Rewards & referrals

Reward eligibility uses **Merkle trees**: eligibility is proven against a published
root rather than stored and checked row by row, which keeps verification cheap and
tamper-evident. A **user-relationship tree** (a graph structure) models the referral
program, so multi-level referral relationships are first-class data.

## Communication & security

Services synchronize over **gRPC** for fast, typed, real-time cross-platform
communication. Access is gated by scoped **JWTs** and **role-based access control**,
and identity is centralized rather than re-implemented per product — fewer places
for auth to drift or break.

## Outcome

The ecosystem got a single identity, a clean wallet abstraction with
account-abstraction UX, and a tamper-evident reward system — delivered as shared
services that every product builds on through one OIDC layer and a gRPC backbone.
